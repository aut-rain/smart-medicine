# 运维手册

## 部署流程

### 后端部署

#### 1. 环境准备
- Java 17+
- MySQL 8.0+
- Redis 6.0+

#### 2. 配置文件
创建 `application-prod.yml` 并配置：
- 数据库连接
- Redis 连接
- AI API 密钥

#### 3. 构建
```bash
cd src
mvn clean package
```

#### 4. 运行
```bash
java -jar target/smart-medicine-1.0.0.jar
```

### 前端部署

#### 1. 构建
```bash
cd smart-medicine-web
npm run build
```

#### 2. 部署到 Nginx
配置反向代理到后端 8080 端口

### Android 发布

#### 1. 生成签名密钥
```bash
keytool -genkey -v -keystore app-release.jks
```

#### 2. 构建发布版
```bash
./gradlew assembleRelease
```

## 监控指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| CPU 使用率 | > 80% | 需要扩容 |
| 内存使用率 | > 85% | 需要扩容 |
| API 响应时间 | > 1000ms | 性能问题 |

## 常见问题

### 后端启动失败
**检查项：**
- MySQL 是否运行
- Redis 是否运行
- 端口是否被占用

### 前端 API 请求失败
**检查项：**
- API 地址配置
- CORS 配置
- 网络代理

### Android 应用闪退
**检查项：**
- Logcat 日志
- 权限配置
- 网络配置

## 回滚流程

### 后端回滚
1. 停止服务
2. 恢复上一个版本
3. 重启服务
4. 验证健康检查

### 前端回滚
1. 备份当前版本
2. 恢复上一版本
3. 重启 Nginx

## 定期维护

- 每日: 检查日志和系统资源
- 每周: 清理过期日志
- 每月: 数据库优化
