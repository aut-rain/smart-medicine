# 运维手册

## 环境配置

### 后端配置文件

配置文件位置: `src/main/resources/config/`

| 配置文件 | 说明 | 关键参数 |
|----------|------|----------|
| `application-db.yml` | 数据库配置 | 数据源URL、用户名、密码 |
| `application-redis.yml` | Redis配置 | 主机、端口、密码 |
| `application-jwt.yml` | JWT安全配置 | 密钥、过期时间 |
| `application-ai.yml` | AI服务配置 | 阿里云通义千问API密钥 |
| `application-oss.yml` | 阿里云OSS配置 | Bucket、密钥 |
| `application-mail.yml` | 邮件服务配置 | SMTP服务器、认证信息 |

### 关键环境变量

| 变量 | 必需 | 说明 | 示例 |
|------|------|------|------|
| `spring.datasource.url` | 是 | 数据库连接URL | `jdbc:mysql://localhost:3306/smart-medicine` |
| `spring.datasource.username` | 是 | 数据库用户名 | `root` |
| `spring.datasource.password` | 是 | 数据库密码 | `********` |
| `spring.ai.dashscope.api-key` | 是 | 阿里云AI密钥 | `sk-*******************` |
| `jwt.secret` | 是 | JWT签名密钥 | 生产环境使用强密钥 |
| `spring.data.redis.host` | 是 | Redis主机 | `localhost` |
| `spring.data.redis.port` | 否 | Redis端口 | `6379` |

### 前端环境变量

文件位置: `smart-medicine-web/.env`

| 变量 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `VITE_API_BASE` | 否 | `http://localhost:8080` | 后端API基础地址 |

### Android网络配置

文件位置: `smart-medicine-Android/app/src/main/java/com/example/smart_medicine_android/di/NetworkModule.kt`

**开发环境**: `http://10.0.2.2:8080` (模拟器)
**生产环境**: 实际服务器地址

## 部署流程

### 后端部署

#### 1. 环境准备
- Java 17+
- MySQL 8.0+
- Redis 6.0+

#### 2. 配置文件
创建 `application-prod.yml` 并配置：
- 数据库连接
- Redis 连接
- AI API 密钥

#### 3. 构建
```bash
cd src
mvn clean package
```

#### 4. 运行
```bash
java -jar target/smart-medicine-1.0.0.jar
```

### 前端部署

#### 1. 构建
```bash
cd smart-medicine-web
npm run build
```

#### 2. 部署到 Nginx
配置反向代理到后端 8080 端口

### Android 发布

#### 1. 生成签名密钥
```bash
keytool -genkey -v -keystore app-release.jks
```

#### 2. 构建发布版
```bash
./gradlew assembleRelease
```

## 常见问题

### 后端启动失败
**检查项：**
- MySQL 是否运行
- Redis 是否运行
- 端口是否被占用
- 日志文件: `logs/smart-medicine.log`

### 前端 API 请求失败
**检查项：**
- API 地址配置 (`VITE_API_BASE`)
- CORS 配置 (`SecurityConfig.java`)
- 网络代理设置

### Android 应用闪退
**检查项：**
- Logcat 日志
- 权限配置 (`AndroidManifest.xml`)
- 网络安全配置 (`network_security_config.xml`)

### 数据库连接失败
**检查项：**
- MySQL 服务状态
- 连接URL格式
- 用户名密码
- 防火墙设置

## 回滚流程

### 后端回滚
1. 停止服务
2. 恢复上一个版本JAR
3. 重启服务
4. 验证健康检查

### 前端回滚
1. 备份当前版本
2. 恢复上一版本
3. 重启 Nginx

### Android回滚
1. 保留旧版本APK
2. 重新安装旧版本
3. 通知用户更新

## 定期维护

- **每日**: 检查日志和系统资源
- **每周**: 清理过期日志，检查磁盘空间
- **每月**: 数据库优化，安全更新
