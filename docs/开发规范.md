## 📐 命名规范

### 1. 包命名
- **强制**: 统一使用小写,点分隔符之间有且仅有一个自然语义的英文单词
- **强制**: 包名统一使用单数形式,类名使用复数形式时,才可以使用复数
- **推荐**: 包名从 `com.medical.smartmedicine` 开始

```java
// ✅ 正确
com.medical.smartmedicine.controller
com.medical.smartmedicine.service.impl
com.medical.smartmedicine.model.dto.user

// ❌ 错误
com.medical.smartMedicine.Controller
world.zzy.service
```

### 2. 类命名
- **强制**: 类名使用 UpperCamelCase 风格
- **强制**: 抽象类命名使用 Abstract 或 Base 开头
- **强制**: 异常类命名使用 Exception 结尾
- **强制**: 测试类命名以要测试的类名开始,以 Test 结尾

```java
// ✅ 正确
public class UserController { }
public abstract class BaseService { }
public class BusinessException extends RuntimeException { }
public class UserServiceTest { }

// ❌ 错误
public class user_controller { }
public class UserC { }
```

### 3. 方法命名
- **强制**: 方法名使用 lowerCamelCase 风格
- **强制**: 获取单个对象的方法用 `get` 做前缀
- **强制**: 获取多个对象的方法用 `list` 做前缀
- **强制**: 获取统计值的方法用 `count` 做前缀
- **强制**: 插入的方法用 `save/insert` 做前缀
- **强制**: 删除的方法用 `remove/delete` 做前缀
- **强制**: 修改的方法用 `update` 做前缀

```java
// ✅ 正确
public UserVO getUserById(Integer id) { }
public List<UserVO> listUsers() { }
public Long countUsers() { }
public User saveUser(User user) { }
public void deleteUser(Integer id) { }
public void updateUser(User user) { }

// ❌ 错误
public UserVO findUser(Integer id) { }  // 应该用get
public List<UserVO> getUsers() { }  // 应该用list
```

### 4. 常量命名
- **强制**: 常量命名全部大写,单词间用下划线隔开
- **强制**: 力求语义表达完整清楚,不要嫌名字长

```java
// ✅ 正确
public static final String USER_DEFAULT_AVATAR = "https://...";
public static final int MAX_STOCK_COUNT = 1000;

// ❌ 错误
public static final String userAvatar = "https://...";
public static final int MAX_COUNT = 1000;
```

### 5. 变量命名
- **强制**: 变量名使用 lowerCamelCase 风格
- **推荐**: 布尔类型的变量,不要加 is,但可以加 has/can/should 等前缀

```java
// ✅ 正确
boolean success;
boolean hasPermission;
boolean canEdit;
String userName;

// ❌ 错误
boolean isSuccess;  // 避免使用is
String user_name;   // 避免下划线
```

## 🏗️ 代码结构规范

### 1. Controller层规范

```java
@RestController
@RequestMapping(ApiConstant.API_VERSION_V1 + "/users")
@Tag(name = "用户管理", description = "用户相关接口")
@RequiredArgsConstructor
public class UserController {

    private final UserService userService;

    @GetMapping("/{id}")
    @Operation(summary = "获取用户详情")
    public Result<UserVO> getUserById(@PathVariable Integer id) {
        UserVO userVO = userService.getUserById(id);
        return Result.success(userVO);
    }

    @PostMapping
    @Operation(summary = "创建用户")
    public Result<UserVO> createUser(@Valid @RequestBody UserCreateRequest request) {
        UserVO userVO = userService.createUser(request);
        return Result.success(userVO);
    }
}
```

**要点**:
- ✅ 使用 `@RestController` 注解
- ✅ 统一路径前缀 `/api/v1`
- ✅ 使用 `@RequiredArgsConstructor` 自动注入
- ✅ 只注入需要的Service,避免注入过多依赖
- ✅ 统一返回 `Result<T>` 类型
- ✅ 参数使用 `@Valid` 校验
- ✅ 添加Swagger注解 `@Tag` 和 `@Operation`

### 2. Service层规范 (接口+实现分离)

#### Service实现
```java
package com.medical.smartmedicine.application.service.impl;

import com.medical.smartmedicine.application.service.UserService;
import com.medical.smartmedicine.common.enums.ResultCode;
import com.medical.smartmedicine.common.exception.BusinessException;
import com.medical.smartmedicine.domain.entity.User;
import com.medical.smartmedicine.domain.vo.UserVO;
import com.medical.smartmedicine.infrastructure.mapper.UserMapper;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
@Slf4j
@RequiredArgsConstructor
public class UserServiceImpl implements UserService {

    private final UserMapper userMapper;
    private final PasswordEncoder passwordEncoder;

    @Override
    @Transactional(rollbackFor = Exception.class)
    public UserVO createUser(RegisterDTO registerDTO) {
        // 1. 参数校验
        if (existsByUserAccount(registerDTO.getUserAccount())) {
            throw new BusinessException(ResultCode.USER_ACCOUNT_EXISTS);
        }

        // 2. 构建Entity
        User user = User.builder()
                .userAccount(registerDTO.getUserAccount())
                .userPwd(passwordEncoder.encode(registerDTO.getUserPwd()))
                .userName(registerDTO.getUserName())
                .build();

        // 3. 保存数据
        userMapper.insert(user);

        // 4. 返回VO
        return convertToVO(user);
    }

    @Override
    public UserVO getUserById(Integer id) {
        User user = userMapper.selectById(id);
        if (user == null) {
            throw new BusinessException(ResultCode.USER_NOT_FOUND);
        }
        return convertToVO(user);
    }

    @Override
    @Transactional(rollbackFor = Exception.class)
    public UserVO updateUser(Integer id, UserUpdateDTO updateDTO) {
        User user = userMapper.selectById(id);
        if (user == null) {
            throw new BusinessException(ResultCode.USER_NOT_FOUND);
        }
        
        // 更新字段
        user.setUserName(updateDTO.getUserName());
        user.setUserAge(updateDTO.getUserAge());
        
        userMapper.updateById(user);
        return convertToVO(user);
    }

    @Override
    @Transactional(rollbackFor = Exception.class)
    public void deleteUser(Integer id) {
        if (userMapper.selectById(id) == null) {
            throw new BusinessException(ResultCode.USER_NOT_FOUND);
        }
        userMapper.deleteById(id);
    }

    /**
     * Entity转VO
     */
    private UserVO convertToVO(User user) {
        return UserVO.builder()
                .id(user.getId())
                .userAccount(user.getUserAccount())
                .userName(user.getUserName())
                // 注意: 不返回密码
                .build();
    }

    /**
     * 检查账号是否存在
     */
    private boolean existsByUserAccount(String userAccount) {
        return userMapper.selectCount(
            new LambdaQueryWrapper<User>()
                .eq(User::getUserAccount, userAccount)
        ) > 0;
    }
}
```

**要点**:
- ✅ Service接口定义在 `application.service` 包
- ✅ Service实现在 `application.service.impl` 包
- ✅ 实现类使用 `@Service` 注解
- ✅ 添加 `@Slf4j` 日志
- ✅ 使用 `@RequiredArgsConstructor` 自动注入
- ✅ 写操作添加 `@Transactional` 注解
- ✅ 抛出 `BusinessException` 而非返回错误码
- ✅ 不返回Entity,返回VO对象
- ✅ 敏感信息不返回(密码等)

### 3. DTO/VO规范

#### Request DTO
```java
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
@Schema(description = "用户创建请求")
public class UserCreateRequest {

    @NotBlank(message = "用户账号不能为空")
    @Pattern(regexp = "^[a-zA-Z0-9]{4,20}$", message = "账号格式错误,只能包含字母数字,长度4-20位")
    @Schema(description = "用户账号")
    private String userAccount;

    @NotBlank(message = "用户密码不能为空")
    @Size(min = 6, max = 20, message = "密码长度为6-20位")
    @Schema(description = "用户密码")
    private String userPwd;

    @NotBlank(message = "用户姓名不能为空")
    @Schema(description = "用户姓名")
    private String userName;

    @Email(message = "邮箱格式不正确")
    @Schema(description = "用户邮箱")
    private String userEmail;
}
```

#### Response VO
```java
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
@Schema(description = "用户信息VO")
public class UserVO {

    @Schema(description = "用户ID")
    private Integer id;

    @Schema(description = "用户账号")
    private String userAccount;

    @Schema(description = "用户姓名")
    private String userName;

    @Schema(description = "用户邮箱")
    private String userEmail;

    @Schema(description = "创建时间")
    @JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")
    private Date createTime;

    // 注意: 不包含密码字段
}
```

**要点**:
- ✅ Request DTO 添加 Jakarta Validation 注解
- ✅ 所有字段添加 `@Schema` 注解
- ✅ VO对象不包含敏感信息
- ✅ 时间字段统一格式 `@JsonFormat`

## ⚠️ 异常处理规范

### 1. 业务异常

```java
// 定义枚举
public enum ResultCode {
    SUCCESS("200", "操作成功"),
    FAIL("400", "操作失败"),
    USER_NOT_FOUND("404", "用户不存在"),
    USER_ACCOUNT_EXISTS("400", "用户账号已存在"),
    INVALID_PASSWORD("400", "密码错误"),
    UNAUTHORIZED("401", "未授权"),
    FORBIDDEN("403", "无权限访问"),
    INTERNAL_SERVER_ERROR("500", "服务器内部错误");

    private final String code;
    private final String message;
}

// 抛出异常
if (user == null) {
    throw new BusinessException(ResultCode.USER_NOT_FOUND);
}

// 自定义消息
throw new BusinessException(ResultCode.FAIL, "用户名或密码错误");
```

### 2. 全局异常处理

```java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public Result<?> handleBusinessException(BusinessException e) {
        log.warn("业务异常: {}", e.getMessage());
        return Result.fail(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<?> handleValidationException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldError().getDefaultMessage();
        return Result.fail(ResultCode.INVALID_PARAM.getCode(), message);
    }

    @ExceptionHandler(Exception.class)
    public Result<?> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.fail(ResultCode.INTERNAL_SERVER_ERROR);
    }
}
```

## 🔒 Spring Security规范

### 1. 权限注解

```java
// 需要登录
@PreAuthorize("isAuthenticated()")
public Result<UserVO> getCurrentUser() { }

// 需要管理员权限
@PreAuthorize("hasRole('ADMIN')")
public Result<Void> deleteUser(@PathVariable Integer id) { }

// 需要特定权限
@PreAuthorize("hasAuthority('user:delete')")
public Result<Void> deleteUser(@PathVariable Integer id) { }

// 自己或管理员可访问
@PreAuthorize("@userSecurity.isSelfOrAdmin(#id)")
public Result<UserVO> getUserById(@PathVariable Integer id) { }
```

### 2. 白名单配置

```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(auth -> auth
            // 公开接口
            .requestMatchers("/api/v1/auth/**").permitAll()
            .requestMatchers("/swagger-ui/**", "/v3/api-docs/**").permitAll()
            // 其他接口需要认证
            .anyRequest().authenticated()
        );
    return http.build();
}
```

## 🧪 测试规范

### 1. 单元测试

```java
@SpringBootTest
class UserServiceTest {

    @Autowired
    private UserService userService;

    @Test
    @DisplayName("创建用户 - 成功")
    void testCreateUser_Success() {
        // Given
        UserCreateRequest request = UserCreateRequest.builder()
            .userAccount("testuser")
            .userPwd("123456")
            .build();

        // When
        UserVO result = userService.createUser(request);

        // Then
        assertNotNull(result);
        assertEquals("testuser", result.getUserAccount());
    }

    @Test
    @DisplayName("创建用户 - 账号已存在")
    void testCreateUser_AccountExists() {
        // Given
        UserCreateRequest request = UserCreateRequest.builder()
            .userAccount("existuser")
            .build();

        // When & Then
        assertThrows(BusinessException.class, () -> {
            userService.createUser(request);
        });
    }
}
```

## 📚 Git提交规范

### 1. 提交格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 2. Type类型

- `feat`: 新功能
- `fix`: 修复Bug
- `docs`: 文档更新
- `style`: 代码格式(不影响功能)
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具相关

### 3. 示例

```bash
# 新功能
git commit -m "feat(user): 添加用户注册功能"

# Bug修复
git commit -m "fix(auth): 修复Token过期判断错误"

# 重构
git commit -m "refactor(controller): 移除BaseController依赖"

# 文档
git commit -m "docs(api): 更新API文档"
```

## 🚫 禁止事项

### 1. 代码层面
- ❌ 禁止使用魔法值(未定义的常量)
- ❌ 禁止在循环中执行数据库操作
- ❌ 禁止捕获异常后不处理(空catch块)
- ❌ 禁止使用System.out.println,使用日志框架
- ❌ 禁止在Controller中写业务逻辑

### 2. 安全层面
- ❌ 禁止明文存储密码
- ❌ 禁止将敏感信息返回给前端
- ❌ 禁止拼接SQL(使用预编译)
- ❌ 禁止忽略异常

### 3. 性能层面
- ❌ 禁止一次性查询大量数据(使用分页)
- ❌ 禁止在循环中调用远程接口
- ❌ 禁止不必要的对象创建
