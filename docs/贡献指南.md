# 贡献指南

感谢您对智慧医疗系统项目的关注！

## 项目概述

智慧医疗系统是一个全栈医疗健康平台，包含：
- **后端**: Spring Boot 3.x + MySQL + Redis
- **Web前端**: React 18 + TypeScript + Vite + Ant Design
- **Android客户端**: Kotlin + Jetpack Compose

## 开发环境准备

### 系统要求

| 工具 | 版本要求 |
|------|----------|
| JDK | 17+ |
| Node.js | 18+ |
| MySQL | 8.0+ |
| Redis | 6.0+ |
| Android Studio | Hedgehog (2023.1.1)+ |

### 克隆项目

```bash
git clone <repository-url>
cd smart-medicine
```

## 快速开始

### 1. 后端启动

```bash
# 进入后端目录
cd src

# 方式一：使用 Maven
mvn spring-boot:run

# 方式二：使用 IDE
# 在 IntelliJ IDEA 中打开 src 目录
# 运行 SmartMedicineApplication.java 的 main 方法
```

**后端端口**: http://localhost:8080
**API文档**: http://localhost:8080/doc.html

### 2. Web前端启动

```bash
# 进入前端目录
cd smart-medicine-web

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```

**前端端口**: http://localhost:5173

### 3. Android客户端

```bash
# 进入 Android 目录
cd smart-medicine-Android

# 构建调试版本
./gradlew assembleDebug

# 安装到设备/模拟器
./gradlew installDebug

# 或使用 Android Studio 打开项目运行
```

详细说明请参考：[smart-medicine-Android/README.md](../smart-medicine-Android/README.md)

## 常用脚本参考

### 后端 (Maven)

| 脚本 | 说明 |
|------|------|
| `mvn spring-boot:run` | 启动开发服务器 |
| `mvn clean package` | 清理并构建 JAR |
| `mvn test` | 运行单元测试 |
| `mvn clean` | 清理构建缓存 |

| 脚本 | 说明 |

## 环境变量配置

### 后端配置文件位置

```
src/main/resources/
├── application.yml           # 主配置文件
└── config/                   # 模块化配置
    ├── application-db.yml    # 数据库配置
    ├── application-redis.yml # Redis配置
    ├── application-jwt.yml   # JWT安全配置
    ├── application-ai.yml    # AI服务配置
    ├── application-oss.yml   # 阿里云OSS配置
    └── application-mail.yml  # 邮件服务配置
```

### Web前端环境变量

创建 `.env` 文件（参考 `.env.example`）：

```bash
# API基础地址
VITE_API_BASE=http://localhost:8080
```

## 开发规范

### 提交规范 (Conventional Commits)

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 添加用户登录功能` |
| `fix` | 修复Bug | `fix: 修复资讯轮播图显示问题` |
| `docs` | 文档更新 | `docs: 更新API文档` |
| `style` | 代码格式调整 | `style: 调整缩进` |
| `refactor` | 重构 | `refactor: 优化Repository层结构` |
| `test` | 测试相关 | `test: 添加用户服务单元测试` |
| `chore` | 构建过程/辅助工具 | `chore: 更新依赖版本` |

### 分支策略

| 分支 | 说明 |
|------|------|
| `main` | 主分支，保持稳定 |
| `develop` | 开发分支 |
| `feature/*` | 功能分支 |
| `bugfix/*` | Bug修复分支 |
